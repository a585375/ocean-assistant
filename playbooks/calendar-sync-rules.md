# Calendar Sync Rules

## 目标
让 Ocean 的时间型任务在 Calendar 中稳定、统一地呈现，并与助理系统保持一致。

## 同步总规则
1. 只要任务带有时间，就同步到 iCloud Calendar
2. 这里的“带时间”包括：
   - 用户明确给出的具体时间
   - 可合理推断的截止时间（例如：下班前 = 18:00）
3. Reminders 不再作为默认同步目标
4. 时间型事项默认在开始前 10 分钟提醒
5. Discord 提醒默认直接 @Ocean

## 时间推断规则
- 下班前：默认记为当天 / 对应日期 18:00
- 上午：如无更精确时间，可根据上下文再判断；不确定时优先询问或仅记 task 不同步 Calendar
- 下午：同上
- 明确到几点：直接按给定时间入 Calendar

## 周任务 / 阶段任务规则
- 周任务默认使用区间型
- Calendar 中应包含：开始时间 + 截止时间
- 默认开始时间：该周一 09:00
- 默认截止时间：该周结束日 18:00，或用户给出的明确节点时间

## 展示原则
- Calendar 用于承接时间型安排
- 助理任务系统保留完整结构化信息
- Calendar 里重点体现时间感和可执行安排，不承担全部知识库职责

## 当前已知策略
- Calendar 已真实打通
- 时间型任务可同步
- 同步时保留 task_id / project / risk / next_action 到 notes 中
