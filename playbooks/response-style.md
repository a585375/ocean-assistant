# Response Style

## 目标
让 Ocean 在不同频道/不同时间问同类问题时，收到的回复格式尽量稳定。

## 任务类回复规则
1. 一个任务 = 一个独立代码块
2. 一个任务的全部信息必须放在同一个代码块里
3. 不要把一个任务拆成多个代码块
4. 多任务之间用正常段落分隔，不要混成一个大代码块

## 风格偏好
- 更像 Claude Code 的干净排版
- 少 emoji、少花哨标签、少废话
- 层级清楚、信息密度高、便于扫读
- 不要重复解释工具过程

## 信息结构建议
每个任务尽量包含：
- 标题
- 时间（如果有）
- 优先级
- 风险
- 项目
- 执行提醒 / 重点注意 / 风险点（比“下一步动作”更实用）

## 同步与提醒规则
- Reminders 不再使用
- 只要任务带有时间（包括可合理推断出的截止时间），就同步到 iCloud Calendar
- 明确时间事项，开始前 10 分钟提醒
- Discord 提醒默认直接 @Ocean
