# 模拟盘 vs 回测 逐笔复盘 Checklist

日期：2026-03-21
项目：AutoTrade
目标：逐笔复盘模拟盘，定位回测与模拟盘差异的根因。

## 复盘目标

把同一时间段内：
- 回测信号
- 回测订单
- 模拟盘信号
- 模拟盘订单/成交
- 风控拦截/过滤

按统一时间线对齐，找出差异出现的第一跳。

---

## 一、先准备材料

### 必备输入
- 模拟盘逐笔成交记录
- 模拟盘订单生命周期记录（提交、修改、撤单、成交）
- 模拟盘运行日志
- 同时间段回测结果
- 回测信号明细
- 回测订单明细
- 同时间段市场数据样本
- 策略配置文件
- 风控参数

### 输出要求
- 至少定位到“第一处出现偏差的环节”
- 不只描述现象，要归因到具体模块/机制
- 结论尽量归类为可修复项 / 不可避免项 / 需进一步验证项

---

## 二、逐层排查顺序

## 1. 时间轴对齐
检查：
- 回测和模拟盘是否使用同一交易日、同一时区、同一 session 定义
- bar 完成时间是否一致
- 日志时间、订单时间、成交时间是否统一到同一参考时钟

重点关注：
- 时区偏移
- 夏令时
- bar close 判定时点
- 延迟导致的信号进入下一根 bar

结论记录：
- 是否存在时间基准不一致
- 偏移了多少秒/多少根 bar

## 2. 数据输入一致性
检查：
- 回测用的行情源与模拟盘用的行情源是否一致
- OHLC / tick / quote / depth 是否同源
- 缺失值、修正值、补齐方式是否不同
- 模拟盘是否用了实时数据，而回测用的是聚合后数据

重点关注：
- NQ 信号输入是否一致
- QQQ / 期权定价输入是否一致
- 历史数据重建方式是否影响信号

结论记录：
- 是否是数据源差异导致
- 差异发生在哪个 symbol / timeframe / 字段

## 3. 信号生成一致性
检查：
- 同一时间点的指标值是否一致
- EMA / ST / ADX / ATR 等核心指标是否一致
- 入场条件是否在同一根 bar 触发
- 是否存在 live only / backtest only 的分支逻辑

重点关注：
- 信号触发时点是否提前/滞后
- 指标 warmup / min_bars 是否一致
- 实时模式与回测模式是否用了不同聚合源

结论记录：
- 第一处信号分叉发生在哪个 timestamp
- 当时的指标值差异是什么

## 4. 风控与过滤条件
检查：
- max trades per day
- cool down
- exit cooldown
- daily max loss
- trading window
- lunch window
- liquidity/spread guard
- quote freshness guard
- underlying session guard

重点关注：
- 回测放行但模拟盘拦截
- 模拟盘放行但回测未实现同等限制

结论记录：
- 是否是风险控制规则导致结果偏差
- 是哪条规则 first impact

## 5. 下单与执行差异
检查：
- 回测撮合模型 vs 模拟盘真实撮合/模拟撮合
- 限价单 aggressive ticks/bps 是否生效
- timeout replace / cancel replace 行为是否仅 live 存在
- bid/ask/mid 作为参考价时是否一致

重点关注：
- 回测假设成交，但模拟盘没成交
- 回测按 bar close 成交，但模拟盘实际挂单后错过
- entry / exit chase 逻辑只在 live 存在

结论记录：
- 是否是执行层差异造成 PnL 偏差
- 偏差主要来自 entry 还是 exit

## 6. 期权链与合约选择差异
检查：
- 回测选中的期权合约与模拟盘是否同一张
- DTE / expiry / strike / right 是否一致
- 动态订阅与预加载逻辑是否影响可选合约范围
- same-day expiry / cutoff / force flatten 是否一致

重点关注：
- 回测用了理论上可选合约，模拟盘因链加载/延迟选了另一张
- quote 不新鲜导致模拟盘跳过，但回测继续成交

结论记录：
- 是否合约选择不同导致结果失真

## 7. 出场与持仓管理差异
检查：
- stop loss / take profit / trailing stop / time stop 是否一致
- quote-driven exit 是否只在 live 生效
- trend exit / trend reentry 是否一致
- protective stop / recovery 机制是否影响模拟盘

重点关注：
- 回测用 bar close 出场，模拟盘按 quote / bid / mid 出场
- 模拟盘受实时 PnL 或流动性保护影响提前退出

结论记录：
- 出场规则是否是主要偏差来源

## 8. 系统恢复与异常处理差异
检查：
- 是否发生过断连 / 重连 / resubscribe
- 是否有 client_id conflict / gateway restart / market data reset
- 这些 runtime patch 是否影响模拟盘行为，而回测没有对应机制

重点关注：
- 模拟盘中途异常导致状态漂移
- 回测是理想连续环境，模拟盘不是

结论记录：
- 是否是系统级运行异常导致差异

## 四、最终目标

不是简单回答“为什么模拟盘没回测那么好”，而是明确：

1. 第一处偏差出现在哪里
2. 哪类差异最影响结果
3. 哪些是系统 bug
4. 哪些是现实交易不可避免的差异
5. 后续应该优先修哪一块
